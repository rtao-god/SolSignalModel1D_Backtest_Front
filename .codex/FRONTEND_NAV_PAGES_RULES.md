# FRONTEND NAV / PAGES RULES 
## Цель
Сохранить единый паттерн навигации и структуры страниц, чтобы не ломать UX и не плодить “мегастраницы”.

## Канонический паттерн (как в PFI/Docs/Main)
1) **Вкладка в Navbar = отдельный домен**  
   Примеры: `Main`, `Docs`, `Analysis`, `Diagnostics`.
2) **Вкладка открывает страницу-карту**  
   Страница-карта содержит **блоки‑ссылки** (карточки) на подстраницы тем.  
   Пример: `DiagnosticsPage` с карточками на `.../backtest`, `.../guardrail`, `.../decisions`, `.../hotspots`.
3) **1 тема = 1 страница**  
   Нельзя держать несколько разных тем на одной странице.  
   Пример: риск/ликвидации = отдельная страница, guardrail = отдельная, решения = отдельная.
4) **Sidebar зависит от вкладки**  
   - Для `Docs` показывать только блоки документов.  
   - Для `Analysis` — только аналитика.  
   - Для `Diagnostics` — только диагностика.  
   Никаких смешанных блоков.
5) **Секции внутри страницы**  
   Если на странице несколько таблиц одной темы — делаем **скролл‑пагинацию** и **якоря** `#section‑N`.
   Подпункты в Sidebar должны вести на эти якоря, как в `PfiPage.tsx`.

## Скролл‑пагинация (как в PFI)
- Секции строятся **динамически из отчёта**, а не захардкожены.
- Для Sidebar нужны подпункты `#section‑N` (или `#diag-section‑N`).
- Секции **мемоизировать**, чтобы hash‑скролл не “залипал”.
- Hash‑скролл применяется **только при первом заходе**, не должен фиксировать пользователя на секции.

## Структура файлов (обязательно)
Для каждой страницы:
- `ui/NamePage.tsx` (только UI/композиция)
- `ui/NamePage.module.scss`
- `ui/types.ts` (все интерфейсы и типы)
 - `index.ts` (реэкспорт только default. Типы — только если реально используются вне папки)

Реальный пример такой структуры папок: diagnosticsPages(это просто тема (вкладка, где и будут лежать главная страница-карта с блоками-ссылками и сами страницы-таблицы))/ui/DiagnosticsPage.tsx(тут как раз лежат главная страница-карта по теме вкладки). Дальше: diagnosticsPages/ui/BacktestDiagnosticsRatingsPage/ui/BacktestDiagnosticsRatingsPage.tsx (тут и рядом с ним на ../ только выше по такой же схеме, лежат уже таблицы и прочее, как раз туда, куда ведут ссылки на главной странице вкладки по темам).

- Если ты не поняла до конца структуру папок, даже после просмотра примеров, то останавливайся и уточни что ты именно не поняла и свои идеи и предположения.

## Разметка названий таблиц
- На фронте НЕ использовать декоративные `===`.
- Заголовки должны быть чистыми и человекочитаемыми.

## Применение по умолчанию
Если задача “добавить таблицы/страницы/вкладки” — следовать этим правилам без доп. согласований.
