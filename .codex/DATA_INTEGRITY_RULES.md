# Правила целостности данных (frontend)

## Основное
- В UI **нельзя** скрывать отсутствие данных.
- Никаких заглушек вида `0`, `false`, `—` вместо реальных значений.
- Если данных нет или они некорректны — **ошибка** должна быть явной и с контекстом (английский текст ошибки).

## Форматирование
- Форматтеры чисел должны **throw**, если значение отсутствует или невалидно.
- UI должен показывать ошибку, а не «красивую» подмену.
